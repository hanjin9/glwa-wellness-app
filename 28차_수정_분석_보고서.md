## 2. 즉시 기술 수정 (지금 바로 진행)

### 2-1. 하단 네비게이션 "미션" 추가

현재 하단 탭 순서: **홈 → 마이 → 쇼핑 → 상담 → 커뮤니티 → VIP**

수정 후 하단 탭 순서: **홈 → 마이 → 미션 → 쇼핑 → 상담 → 커뮤니티 → VIP** (7개)

미션 탭 클릭 시 오늘의 미션 페이지(`/missions`)로 바로 이동합니다.

### 2-2. 멤버십 등급 Green Emerald, Blue Diamond 추가

현재 8등급 체계에서 이미 Green Emerald와 Blue Diamond가 DB 스키마에 정의되어 있습니다. 프론트엔드 멤버십 페이지에서 누락된 부분을 확인하고 보완합니다.

| 등급 | 영문명 | 색상 테마 |
|------|--------|----------|
| 1 | Silver LEVEL | 은색 |
| 2 | Gold LEVEL | 금색 |
| 3 | Blue Sapphire LEVEL | 파란 사파이어 |
| 4 | **Green Emerald LEVEL** | 초록 에메랄드 |
| 5 | Diamond LEVEL | 다이아몬드 |
| 6 | **Blue Diamond LEVEL** | 파란 다이아몬드 |
| 7 | Platinum LEVEL | 플래티넘 |
| 8 | Black Platinum LEVEL | 블랙 플래티넘 |

### 2-3. 멤버십 등급 관리자만 변경 가능

현재 문제: 고객이 직접 "등급 업그레이드" 버튼을 눌러서 자기 등급을 변경할 수 있음

수정 방향:
- 고객 화면에서 "등급 업그레이드" 버튼 제거
- 관리자 전용 등급 변경 API 구현 (adminProcedure)
- 자동 승급 알고리즘: 포인트 누적 + 미션 달성률 + 참여도 기준 충족 시 자동 승급

자동 승급 알고리즘 기본 설계:

| 등급 | 필요 포인트 | 미션 달성률 | 참여 기간 |
|------|-----------|-----------|----------|
| Silver | 0 | - | 가입 즉시 |
| Gold | 1,000 | 50% | 1개월 |
| Blue Sapphire | 5,000 | 60% | 3개월 |
| Green Emerald | 15,000 | 70% | 6개월 |
| Diamond | 30,000 | 75% | 1년 |
| Blue Diamond | 60,000 | 80% | 2년 |
| Platinum | 100,000 | 85% | 3년 |
| Black Platinum | 관리자 지정 | 관리자 지정 | 관리자 지정 |

### 2-4. 커뮤니티 명예의전당 → VIP 라운지

커뮤니티 전체 카테고리 맨 오른쪽의 "명예의전당"을 "VIP 라운지"로 명칭 변경합니다.

### 2-5. 다국어 번역 전체 페이지 적용

현재 일부 페이지에만 다국어가 적용되어 있습니다. 모든 페이지의 하드코딩된 한국어 텍스트를 다국어 시스템으로 전환합니다.

### 2-6. 외부 링크 인앱 브라우저 처리

현재 문제: 네이버 날씨, 코인 차트 등 외부 링크 클릭 시 앱에서 완전히 빠져나감

수정 방향:
- 웹 버전: 새 탭(`target="_blank"`)으로 열기 + "돌아가기" 안내
- 앱 버전(Capacitor): `Browser.open()` API로 인앱 브라우저 사용 → 닫으면 자동으로 앱으로 복귀

### 2-7. 건강 진단 결과 페이지 별도 생성

현재 문제: "건강 진단 결과" 클릭 시 체크하는 화면(인체 모양)으로 이동

수정 방향:
- **건강 체크 페이지** (`/health-check`): 인체 모양 + 체크리스트 (입력용)
- **건강 진단 결과 페이지** (`/health-results`): AI 분석 결과 + 코치 평가 (결과 조회용)

건강 진단 결과 페이지 구성:

| 섹션 | 내용 |
|------|------|
| AI 종합 분석 | 건강 점수, 위험도, 개선 권장사항 |
| 코치 평가 | 전문 코치의 추가 코멘트 (관리자가 입력) |
| 부위별 상태 | 인체 이미지에 색상으로 상태 표시 |
| 추천 미션 | AI가 분석 결과 기반으로 추천하는 미션 |
| 이전 결과 비교 | 지난 진단과 비교 그래프 |

## 4. GitHub 벤치마킹 결과 - AI 알고리즘 적용 방안

### 4-1. 건강 데이터 AI 분석 (OpenHealth 참고)

OpenHealth 프로젝트(GitHub Stars 3.8k)의 접근 방식을 참고하여, 사용자의 건강 데이터를 LLM에 전달하고 개인화된 건강 분석 결과를 생성합니다. 우리 앱에는 이미 `invokeLLM` 헬퍼가 내장되어 있으므로, 별도 API 키 없이 바로 사용할 수 있습니다.

적용 방식:
1. 사용자의 건강 체크 데이터(혈압, 혈당, 체중, 수면, 스트레스, 통증, 식사, 운동) 수집
2. 인체 부위별 체크 데이터 포함
3. LLM에 구조화된 프롬프트로 전달
4. JSON 스키마로 구조화된 분석 결과 수신
5. 건강 진단 결과 페이지에 표시

### 4-2. 미션 시스템 알고리즘 (Oasis + Loop Habit Tracker 참고)

Oasis 게이미피케이션 프레임워크의 PBML(Points, Badges, Milestones, Leaderboards) 모델과 Loop Habit Tracker의 습관 강도 계산 공식을 참고합니다.

미션 점수 계산 공식:

```
미션 점수 = 기본 점수 × 난이도 가중치 × 기간 보너스 × 스트릭 보너스

기본 점수: 미션 완료 시 100점
난이도 가중치: 쉬움(1.0), 보통(1.5), 어려움(2.0)
기간 보너스: 일간(1.0), 주간(1.2), 이주간(1.5), 월간(2.0), 분기(3.0), 연간(5.0)
스트릭 보너스: 연속 달성 일수 × 0.1 (최대 2.0)
```

건강 점수 계산 공식:

```
건강 점수 = Σ(각 지표 점수 × 가중치) / Σ(가중치)

지표별 가중치:
- 혈압: 0.15
- 혈당: 0.15
- BMI: 0.10
- 운동량: 0.15
- 수면: 0.15
- 스트레스: 0.10
- 식사 규칙성: 0.10
- 통증 수준: 0.10
```
